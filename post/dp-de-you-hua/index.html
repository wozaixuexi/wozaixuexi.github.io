对于1D/1D的动态规划的优化

2020-05-22

前言

众所周知，dp可以用来解决一些最优化问题，即求最大值或最小值。

那么现在，我们有一种模型，其状态转移方程如下所示：

f[i]=min{f[j]+w(j,i)} 或 f[i]=max{f[j]+w(j,i)}f[i]=min\{f[j]+w(j,i)\} \ 或\ f[i]=max\{f[j]+w(j,i)\} f[i]=min{f[j]+w(j,i)} 或 f[i]=max{f[j]+w(j,i)}

其中w(j,i)w(j,i)w(j,i)表示选择jjj对f[i]f[i]f[i]产生的贡献（看不明白没关系，可以结合下文理解）。

对于上式所代表的模型，我们将其称为1D/1D的动态规划。

接下来的内容，便是讨论如何对1D/1D的动态规划进行优化。

单调队列优化

【例】LuoguP3572 [POI2014]PTA-Little Bird

题意：从左到右共有nnn根柱子，每根柱子都有各自的高度。有一只鸟，从第111根柱子出发，要跳到第nnn根柱子。若跳到的柱子的高度大于等于当前柱子的高度，则需要111点代价，否则无需代价。给出qqq次询问，每次询问在跳一次最多只能跳过kkk根柱子的情况下，跳到第nnn根柱子所需的最小代价。
数据范围：2≤n≤1062 \leq n \leq 10^62≤n≤106 1≤q≤251 \leq q \leq 251≤q≤25

观察数据范围，发现qqq很小，我们对每一次询问分别求解即可。现在我们考虑如何处理单个询问。

设f[i]f[i]f[i]为跳到第iii根柱子所需的最小代价，转移时枚举最后一次跳是从那一根柱子跳过来的，那我们就有状态转移方程：

f[i]=min⁡i−k≤j≤i−1{f[j]+[hj≤hi]}f[i]=\min \limits_{i-k \leq j \leq i-1} \{f[j]+[h_j \leq h_i] \} f[i]=i−k≤j≤i−1min​{f[j]+[hj​≤hi​]}

显而易见，这是个1D/1D的动态规划。这里的[hj≤hi][h_j \leq h_i][hj​≤hi​]对应的就是上文提到的w(j,i)w(j,i)w(j,i)。

然而这一个dp是O(n2)O(n^2)O(n2)的，n2n^2n2并不能过百万，所以我们需要优化。

可以发现，复杂度之所以这么高，是因为我们对于每一个f[i]f[i]f[i]都枚举了所有可能的决策点jjj，但是我们真的有必要枚举所有的jjj吗？

先下个定义，如果一个决策点jjj在以后的转移中不可能成为最优决策点，或可以被其它同样优的决策点替代，那我们就称jjj为不必要的，否则称为必要的。显然，对于任何一个不必要的jjj，我们都可以将其舍弃。

那么，如何判断一个jjj是不必要的呢？

考虑已经有决策集合SSS，即所有必要的决策点组成的集合。现在新计算出了f[i]f[i]f[i]，要将决策点iii加入决策集合SSS中。在加入iii的同时，SSS中的一些决策点将会变成不必要的。为了排除那些不必要的决策点，我们可以这样考虑：

  • 设w(j,i)=[hj≤hi]w(j,i)=[h_j \leq h_i]w(j,i)=[hj​≤hi​]。
  • 若f[j]>f[i]f[j]>f[i]f[j]>f[i]，则决策点jjj是不必要的。 - 这是因为，对于大于iii的任意一个ppp，选择jjj的代价为f[j]+w(j,p)f[j]+w(j,p)f[j]+w(j,p)，选择iii的代价为f[i]+w(i,p)f[i]+w(i,p)f[i]+w(i,p)。可以发现，当f[j]>f[i]f[j]>f[i]f[j]>f[i]时，无论w(j,p)w(j,p)w(j,p)和w(i,p)w(i,p)w(i,p)怎么取值（只能是000或111），都不会有f[j]+w(j,p)<f[i]+w(i,p)f[j]+w(j,p)<f[i]+w(i,p)f[j]+w(j,p)<f[i]+w(i,p)。决策点iii将比jjj更优或一样优，所以此时jjj是不必要的，我们可以将其排除。
  • 若f[j]=f[i]f[j]=f[i]f[j]=f[i]且hj≤hih_j\leq h_ihj​≤hi​，则决策点jjj是不必要的。 - 同样的，我们考虑对于大于iii的任意一个ppp，选择jjj的代价为f[j]+w(j,p)f[j]+w(j,p)f[j]+w(j,p)，选择iii的代价为f[i]+w(i,p)f[i]+w(i,p)f[i]+w(i,p)。因为hj≤hih_j\leq h_ihj​≤hi​，所以w(j,k)≥w(i,p)w(j,k)\geq w(i,p)w(j,k)≥w(i,p)，所以必有f[j]+w(j,p)≥f[i]+w(i,p)f[j]+w(j,p)\geq f[i]+w(i,p)f[j]+w(j,p)≥f[i]+w(i,p)。那么我们就可以将不必要的jjj排除。
  • 其他的情况都无法将jjj排除。

具体实现时，我们可以使用单调队列这一数据结构来维护当前的决策集合。流程如下：

  • 检查队头，若队头的决策点已经小于i−ki-ki−k，即超过限制范围，则出队，直到队头决策点合法为止。
  • 取队头决策点为当前决策点，计算f[i]f[i]f[i]。
  • 检查队尾，将队尾决策点与iii相比较，排除不必要的决策点。
  • 将iii从队尾入队。

不难发现，在单调队列中，f[j]f[j]f[j]是单调不降的，对于一段相同的f[j]f[j]f[j]，hjh_jhj​是单调下降的（类似于双关键字排序）。这是为什么我们能用单调队列维护的原因，也是为什么队头决策点为当前最优决策点的原因。（可能有点混乱，请自行在实践中理解。）

这样的话，每一个决策点最多只会入队一次，出队一次，转移是O(1)O(1)O(1)的，那么处理单次询问的复杂度就是O(n)O(n)O(n)，总复杂度即为O(qn)O(qn)O(qn)，可以通过此题。

POJ1821 Fence

题意：从左到右共有nnn块木板，现在有kkk个工匠要粉刷这些木板，木板不能重复刷。第iii名工匠可以选择不粉刷任何木板，也可以选择粉刷一段连续的木板，这一段木板必须包含第SiS_iSi​块且长度不超过LiL_iLi​，每刷一块木板工匠即可获得PiP_iPi​的报酬。求工匠们可以获得的最大总报酬是多少。
数据范围：1≤n≤160001 \leq n \leq 160001≤n≤16000 1≤k≤1001 \leq k \leq 1001≤k≤100

设f[i][j]f[i][j]f[i][j]表示前iii个工匠，前jjj块木板，此时的最大总报酬是多少。

首先，应该把工匠按SiS_iSi​从小到大排序，这样以后，每一个工匠刷的木板就会有序，即每一个工匠粉刷的木板一定是在上一个工匠粉刷的木板之后，以便于我们dp。

于是我们就有状态转移方程：

f[i][j]=max{f[i][j−1],f[i−1][j],max⁡j−Li≤l≤Si−1{f[i−1][l]+(j−l)⋅Pi}}f[i][j]=max\{f[i][j-1],f[i-1][j],\max\limits_{j-L_i \leq l \leq S_i-1}\{f[i-1][l]+(j-l)\cdot P_i\}\} f[i][j]=max{f[i][j−1],f[i−1][j],j−Li​≤l≤Si​−1max​{f[i−1][l]+(j−l)⋅Pi​}}

然而这是O(kn2)O(kn^2)O(kn2)的，需要优化。

可以发现，f[i][j−1]f[i][j-1]f[i][j−1]和f[i−1][j]f[i-1][j]f[i−1][j]可以O(1)O(1)O(1)转移到f[i][j]f[i][j]f[i][j]，我们着重处理第三部分。首先那个(i−l)⋅Pi(i-l)\cdot P_i(i−l)⋅Pi​很讨厌，与jjj和lll都有关。我们可以拆一下。

f[i][j]=max{f[i][j−1],f[i−1][j],max⁡j−Li≤l≤Si−1{f[i−1][l]−l⋅Pi}+j⋅Pi}f[i][j]=max\{f[i][j-1],f[i-1][j],\max\limits_{j-L_i \leq l \leq S_i-1}\{f[i-1][l]-l\cdot P_i\}+j\cdot P_i\} f[i][j]=max{f[i][j−1],f[i−1][j],j−Li​≤l≤Si​−1max​{f[i−1][l]−l⋅Pi​}+j⋅Pi​}

这样就很舒服了。

仍然考虑什么样的决策点是非必要的。现在新加入一个决策点jjj，显而易见，若f[i−1][l]−l⋅Pi≤f[i−1][j]−j⋅Pif[i-1][l]-l\cdot P_i \leq f[i-1][j]-j\cdot P_if[i−1][l]−l⋅Pi​≤f[i−1][j]−j⋅Pi​，则决策点lll是非必要的。这是因为有长度LiL_iLi​这个限制，决策点jjj比决策点lll更靠后，决策点jjj能贡献的范围就更大，若决策点jjj本身的贡献还大于等于lll的贡献，那lll就没有必要存在了。就像那句老话说的：

如果一个人比你小还比你强，那你就真的打不过他了。

总结：应学会拆分决策点的贡献。

仍然使用单调队列实现，实现细节参照代码。时间复杂度：O(kn)O(kn)O(kn)。

Code:

#include<iostream>
#include<cstdio>
#include<algorithm>
    using namespace std;
struct Wk {int L,P,S;} s[20005];
    int q[20005],f[105][20005];
bool cmp(Wk x,Wk y) {return x.S<y.S;}
int GetVal(int i,int j)
    {return f[i-1][j]-j*s[i].P;}
int main()
{
    int n=0,k=0;
    scanf("%d%d",&n,&k);
    for(int i=1;i<=k;i++)
        scanf("%d%d%d",&s[i].L,&s[i].P,&s[i].S);
    sort(s+1,s+k+1,cmp);
    for(int i=1;i<=k;i++)
    {
        int L=1,R=0;
        q[++R]=0;
        for(int j=1;j<=n;j++)
        {
            f[i][j]=max(f[i-1][j],f[i][j-1]);
            while(L<=R&&j-q[L]>s[i].L) L++;
            if(j<s[i].S)
            {
                while(L<=R&&GetVal(i,j)>=GetVal(i,q[R])) R--;
                q[++R]=j;
            }
            else if(L<=R)
                f[i][j]=max(f[i][j],s[i].P*j+GetVal(i,q[L]));
        }
    }
    printf("%d",f[k][n]);
    return 0;
}

POJ3017 Cut the Sequence

题意：有一个长度为nnn的序列，现在要把这个序列分成连续的几段，每一段中所有数的和都不能超过mmm。现在要使每段中所有数的最大值之和最小，输出这个最小值。
数据范围：1≤n≤1051 \leq n \leq 10^51≤n≤105 0≤a[i]≤1060\leq a[i]\leq 10^60≤a[i]≤106

设f[i]f[i]f[i]表示前iii个数的答案（即要求的”最小值“）,于是就有状态转移方程：$$f[i]=\min\limits_{0 \leq j \leq i-1 \ and \ \sum\limits_{k=j+1}^i a[k] \leq m}{f[j]+\max\limits_{j+1 \leq k \leq i}a[i]}$$

然而这是O(n2)O(n^2)O(n2)的，所以我们需要优化。

这个式子比较烦，又是求和又是求最大值，也不好拆开，不过我们还是按照套路，考虑什么样的决策点是非必要的（或必要的）。

先设w(j,i)=max⁡j+1≤k≤ia[k]w(j,i)=\max\limits_{j+1 \leq k \leq i}a[k]w(j,i)=j+1≤k≤imax​a[k]。

首先，可以注意到fff数组是单调不降的。

那么考虑一个决策点jjj，对于所有小于jjj的决策点kkk，都有f[k]≤f[j]f[k]\leq f[j]f[k]≤f[j]。若我们要使jjj不被kkk淘汰，我们就要保证f[k]+w(k,i)>f[j]+w(j,i)f[k]+w(k,i)>f[j]+w(j,i)f[k]+w(k,i)>f[j]+w(j,i)，则至少需要满足w(k,i)>w(j,i)w(k,i)> w(j,i)w(k,i)>w(j,i)。

显然，w(k,i)w(k,i)w(k,i)囊括的范围包含了w(j,i)w(j,i)w(j,i)，所以必然有w(k,i)≥w(j,i)w(k,i)\geq w(j,i)w(k,i)≥w(j,i)，关键是什么时候等号成立。

可以发现，当{a[k+1],a[k+2],⋅⋅⋅,a[i]}\{a[k+1],a[k+2],···,a[i]\}{a[k+1],a[k+2],⋅⋅⋅,a[i]}中的最大值位于{a[j+1],a[j+2],⋅⋅⋅,a[i]}\{a[j+1],a[j+2],···,a[i]\}{a[j+1],a[j+2],⋅⋅⋅,a[i]}时，等号成立。

所以，当a[j]=max⁡j≤l≤ia[l]a[j]=\max\limits_{j \leq l \leq i}a[l]a[j]=j≤l≤imax​a[l]成立时，一定有w(k,i)>w(j,i)w(k,i)> w(j,i)w(k,i)>w(j,i)，因为此时{a[k+1],a[k+2],⋅⋅⋅,a[i]}\{a[k+1],a[k+2],···,a[i]\}{a[k+1],a[k+2],⋅⋅⋅,a[i]}中的最大值位于{a[k+1],a[k+2],⋅⋅⋅,a[j]}\{a[k+1],a[k+2],···,a[j]\}{a[k+1],a[k+2],⋅⋅⋅,a[j]}中。（因为aaa数组中可能存在相同的数，所以当上述条件）
