首先，我们可以观察样例解释：

时间 用户操作 人民币(元) A 券的数量 B 券的数量
开户 无 100 0 0
第一天 买入100元 0 50 50
第二天 卖出100% 150 0 0
第二天 买入150元 0 75 37.5
第三天 卖出100% 225 0 0

可以发现，给出的这一种最优方案中，每一次买都会把所有钱都用完，每一次卖都会把所有券都卖光。
于是我们可以领悟到，若要使最后得到的金钱数目最多，**每一次买入都应用光所有钱，每一次卖出都应卖出100%。**这可以通过贪心的方式证明，此处不再赘述。

这启发我们，设f[i]f[i]f[i]表示前iii天最多可以获得多少金钱，考虑第iii天的操作。

首先，第iii天可以不进行任何操作：f[i]=f[i−1]f[i]=f[i-1]f[i]=f[i−1]。

然后我们可以在第iii天卖出当前拥有的金券，这时便需要我们枚举上一次买入金券是什么时候。

我们也可以在这一天买入金券，但这并不会使前iii天的答案增大，故无需考虑。（对于这一天买入对未来的影响，将会在后面的天数中考虑到。）

重点考虑如何处理“卖出”的情况。

对于f[i]f[i]f[i]，我们枚举在第iii天之前的第jjj天，第jjj天能获得的最多金钱数为f[j]f[j]f[j]。考虑f[j]f[j]f[j]这么多的钱在第jjj天能买到多少AAA券、多少BBB券。

设总共能买到kkk张AAA券和BBB券，则：

AAA券有k⋅RatejRatej+1\dfrac{k\cdot Rate_j}{Rate_j+1}Ratej​+1k⋅Ratej​​张。

BBB券有kRatej+1\dfrac{k}{Rate_j+1}Ratej​+1k​。

花费为k⋅Ratej⋅AjRatej+1+k⋅BjRatej+1=k⋅Ratej⋅Aj+BjRatej+1\dfrac{k\cdot Rate_j\cdot A_j}{Rate_j+1}+\dfrac{k\cdot B_j}{Rate_j+1}=k\cdot\dfrac{Rate_j\cdot A_j+B_j}{Rate_j+1}Ratej​+1k⋅Ratej​⋅Aj​​+Ratej​+1k⋅Bj​​=k⋅Ratej​+1Ratej​⋅Aj​+Bj​​。

于是有f[j]=k⋅Ratej⋅Aj+BjRatej+1⇒k=f[j]⋅Ratej+1Ratej⋅Aj+Bjf[j]=k\cdot\dfrac{Rate_j\cdot A_j+B_j}{Rate_j+1}\Rightarrow k=f[j]\cdot \dfrac{Rate_j+1}{Rate_j\cdot A_j+B_j}f[j]=k⋅Ratej​+1Ratej​⋅Aj​+Bj​​⇒k=f[j]⋅Ratej​⋅Aj​+Bj​Ratej​+1​。

于是AAA券有f[j]⋅RatejRatej⋅Aj+Bjf[j]\cdot \dfrac{Rate_j}{Rate_j\cdot A_j+B_j}f[j]⋅Ratej​⋅Aj​+Bj​Ratej​​张，BBB券有f[j]⋅1Ratej⋅Aj+Bjf[j]\cdot \dfrac{1}{Rate_j\cdot A_j+B_j}f[j]⋅Ratej​⋅Aj​+Bj​1​张。

为了方便，设Pj=f[j]⋅RatejRatej⋅Aj+BjP_j=f[j]\cdot \dfrac{Rate_j}{Rate_j\cdot A_j+B_j}Pj​=f[j]⋅Ratej​⋅Aj​+Bj​Ratej​​，Qj=f[j]⋅1Ratej⋅Aj+BjQ_j=f[j]\cdot \dfrac{1}{Rate_j\cdot A_j+B_j}Qj​=f[j]⋅Ratej​⋅Aj​+Bj​1​。

那么我们就有状态转移方程：

f[i]=max⁡1≤j≤i−1{Pj⋅Ai+Qj⋅Bi}f[i]=\max\limits_{1\leq j\leq i-1}\{P_j\cdot A_i+Q_j\cdot B_i\} f[i]=1≤j≤i−1max​{Pj​⋅Ai​+Qj​⋅Bi​}

这样dp是O(n2)O(n^2)O(n2)的，观察到存在与iii和jjj均有关的乘积项，考虑斜率优化。

去掉maxmaxmax。

f[i]=Pj⋅Ai+Qj⋅Bif[i]=P_j\cdot A_i+Q_j\cdot B_i f[i]=Pj​⋅Ai​+Qj​⋅Bi​

似乎很难变成斜率优化的一般形式。我们可以两边同时除以BiB_iBi​。

f[i]Bi=AiBi⋅Pj+Qj\dfrac{f[i]}{B_i}=\dfrac{A_i}{B_i}\cdot P_j+Q_j Bi​f[i]​=Bi​Ai​​⋅Pj​+Qj​

移项：

Qj=−AiBi⋅Pj+f[i]BiQ_j=-\dfrac{A_i}{B_i}\cdot P_j+\dfrac{f[i]}{B_i} Qj​=−Bi​Ai​​⋅Pj​+Bi​f[i]​

现在这条式子就可以看成是一条斜率为−AiBi-\dfrac{A_i}{B_i}−Bi​Ai​​的直线了。

但当我们要进行优化的时候，发现斜率−AiBi-\dfrac{A_i}{B_i}−Bi​Ai​​和横坐标PjP_jPj​都没有单调性，难以优化。

这可以用平衡树动态维护凸壳来解决，但也可以使用较为简单的CDQCDQCDQ分治来求解。

用CDQCDQCDQ分治求解的大致步骤：

  • 将所有决策点（从1到n）存入数组p中。

  • ppp中的点按斜率从大到小排序。

  • 开始分治求解：

  • 设当前处理的决策点范围为l到r，对应p数组中的·第fl到第fr个元素。

  • 设mid=⌊l+r2⌋mid=\lfloor \dfrac{l+r}{2} \rfloormid=⌊2l+r​⌋，将p数组中的决策点以mid为界限分到左右两旁。

  • 递归计算l到mid。

  • 左侧的点已按横坐标从小到大排序（见最后一步），用单调队列直接加点维护凸壳。

  • 右侧的点斜率依然单调，可以快速从单调队列维护的凸壳中寻找最优决策点，从而更新答案。

  • 递归计算mid+1到r。

  • 将p中的点按横坐标从小到大排序，方便上一层处理（可以使用归并排序）。

    如果使用归并排序，时间复杂度应为O(nlogn)O(nlogn)O(nlogn)，可以通过此题。具体细节看代码。

    （注：不要忘了在分治底层用f[i−1]f[i-1]f[i−1]来更新f[i]f[i]f[i]。）

#include<iostream>
#include<cstdio>
#include<algorithm>
	using namespace std;
	const int N=100005;
	int n=0,S=0,p[N],t[N],q[N];
	double A[N],B[N],Ra[N],f[N];
double X(int x)
	{return Ra[x]/(Ra[x]*A[x]+B[x])*f[x];}
double Y(int x)
	{return 1/(Ra[x]*A[x]+B[x])*f[x];}
bool cmp0(int x,int y)
	{return -A[x]/B[x]>-A[y]/B[y];}
bool cmp1(int x,int y)
	{return X(x)<X(y);}
double CalcXL(int k1,int k2)
{
	if(X(k2)==X(k1)) return 9e18; 
	return (Y(k2)-Y(k1))/(X(k2)-X(k1));
}
void PaiXv(int l,int mid,int r)//归并排序
{
	int p1=l,p2=mid+1,cnt=0;
	while(p1<=mid||p2<=r)
	{
		if(p1<=mid&&(p2>r||X(p[p1])<X(p[p2])))
			t[++cnt]=p[p1++];
		else t[++cnt]=p[p2++];
	}
	for(int i=l;i<=r;i++) p[i]=t[i-l+1];
}
void Sol(int l,int r,int fl,int fr)//分治
{
	if(l==r) 
		{f[l]=max(f[l],f[l-1]);return;}
	int mid=(l+r)>>1,cnt=0;
	for(int i=fl;i<=fr;i++)
		if(p[i]<=mid) t[++cnt]=p[i];
	int fmid=fl+cnt-1;
	for(int i=fl;i<=fr;i++)
		if(p[i]>mid) t[++cnt]=p[i];
	for(int i=fl;i<=fr;i++) p[i]=t[i-l+1];
	Sol(l,mid,fl,fmid);
	int L=1,R=0;
	for(int i=fl;i<=fmid;i++)
	{
		while(R>=2&&CalcXL(q[R-1],q[R])<CalcXL(q[R],p[i])) R--;
		q[++R]=p[i];
	}
	for(int i=fmid+1;i<=fr;i++)
	{
		while(R-L+1>=2&&CalcXL(q[L],q[L+1])>-A[p[i]]/B[p[i]]) L++;
		f[p[i]]=max(f[p[i]],X(q[L])*A[p[i]]+Y(q[L])*B[p[i]]);
	}
	Sol(mid+1,r,fmid+1,fr);
	PaiXv(fl,fmid,fr);
}
int main()
{
	scanf("%d%d",&n,&S);
	for(int i=1;i<=n;i++)
		scanf("%lf%lf%lf",&A[i],&B[i],&Ra[i]),p[i]=i;
	sort(p+1,p+n+1,cmp0);
	f[1]=S,Sol(1,n,1,n); 
	printf("%.3f",f[n]);
	return 0;
}
